v0.2.0 June 2026

@vanduo-oss/vue

Changed
  • Decoupled theme defaults from the docs site

    The package now ships the generic Vanduo baseline from @vanduo-oss/core instead of baking in the vd2 site override (PRIMARY_DARK: "blue"). Apps supply their own overrides via setThemeDefaults(), or app.use(VanduoVue, { themeDefaults }).

  • Theme defaults API

    Added setThemeDefaults(overrides), getThemeDefaults(), and the ThemeDefaults type. VanduoVue accepts a themeDefaults option.

Breaking
  • Baked-in dark primary removed

    Consumers that relied on the default PRIMARY_DARK: "blue" must pass { themeDefaults: { PRIMARY_DARK: "blue" } } to restore it. The internal DEFAULTS export was removed — use getThemeDefaults().

v0.1.0 June 2026 Initial Release

@vanduo-oss/vue

New
  • Initial Vue 3 component library

    41 Vd* components (including 7 layout primitives) and 30 composables, extracted from the docs app and published as @vanduo-oss/vue.

  • VanduoVue plugin

    app.use(VanduoVue) plus loadVanduoRuntime() to bootstrap the framework's client-side JS engine alongside Vue.

  • CSS entry point

    @vanduo-oss/vue/css forwards the Vanduo stylesheet for drop-in styling.
