v1.3.1 March 20, 2026

Framework

Critical Security Fixes
  • Suggest — XSS in renderItems()

    renderItems() now escapes text through _escapeHtml() before applying the highlight regex via innerHTML. Previously, raw server/user data was inserted directly — a malicious suggestion could execute arbitrary scripts.

  • Select — 3 broken querySelector selectors fixed

    Selector .vd-custom-select-option corrected to .custom-select-option in updateSelectedOptions(), openDropdown(), and handleKeydown(). Keyboard navigation and programmatic updates were silently failing.

  • Select — generateId() now assigns element.id

    generateId() assigns the generated ID to element.id before returning. Previously ARIA aria-labelledby pointed to a nonexistent ID, breaking screen-reader label association.

Medium Severity Fixes
  • Code Snippet — codesnippet:copy event reports actual success

    The codesnippet:copy custom event now passes the real copySuccess result instead of a hardcoded true.

  • Modals — single shared ESC listener replaces O(n) per-modal handlers

    A single shared keydown handler on document replaces per-modal listeners. Handler is installed on first modal init and removed by destroyAll().

  • Dropdown + Select — typeahead buffer moved to per-instance state

    _typeaheadBuffer and _typeaheadTimer are now stored per-instance in the instances Map. Previously, typing in one dropdown corrupted the typeahead state of another.

  • Navbar — CSS class body-navbar-open replaces inline overflow:hidden

    Replaced document.body.style.overflow = 'hidden' with the CSS class .body-navbar-open. Prevents the navbar and modal scroll locks from clobbering each other.

  • Helpers — on() delegation returns the wrapper for off()

    on() now returns the bound wrapper function so callers can pass it to off() to remove delegation listeners.

  • Validate — 100-char limit on user regex patterns (ReDoS prevention)

    User-supplied regex patterns in the pattern rule are now capped at 100 characters to prevent catastrophic backtracking.

  • Validate — match rule uses CSS.escape() on param

    The match rule now applies CSS.escape() to the param before building the querySelector, preventing selector injection. Wrapped in try/catch for invalid selectors.

Low Severity Fixes
  • Image Box — addEventListener('load', {once:true}) replaces DOM0 onload

    Replaced this.img.onload = ... with addEventListener('load', ..., { once: true }). Handler is cleaned up in close() if the image hasn't loaded yet.

  • Theme Customizer — removed redundant second applyTheme() in reset()

    The duplicate this.applyTheme(this.DEFAULTS.THEME) call in reset() has been removed.

Ecosystem

    New experimental vd-hex interactive canvas grid (selection, pan/zoom, terrain helpers, rotation). Full ESM source, modules, and README are published separately at github.com/vanduo-oss/labs (hex-grid.js, utils/hex-math.js).

v1.1.0 February 7, 2026 Breaking Changes

Framework

  • Breaking: CSS Class Prefixing

    All CSS classes now use the vd- prefix to prevent collisions with other frameworks. Examples: .btn → .vd-btn, .container → .vd-container, .card → .vd-card.

  • Breaking: Manual Initialization Required

    Framework no longer auto-initializes on page load. You must explicitly call Vanduo.init() after loading the script. This allows better control over initialization timing in SPAs and complex applications.

  • New: Multiple Module Formats

    Added ESM (vanduo.esm.js) and CJS (vanduo.cjs.js) build outputs alongside IIFE format. Modern bundlers can now tree-shake unused components.

  • New: Package.json Exports Map

    Added proper exports field in package.json for clean imports: import { Vanduo } from '@vanduo-oss/framework'.

  • Improvement: CSS Namespace Isolation

    The vd- prefix ensures Vanduo can be safely embedded in applications using Bootstrap, Tailwind, or other CSS frameworks without class name collisions.
